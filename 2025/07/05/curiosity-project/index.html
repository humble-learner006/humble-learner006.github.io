curiosity project

在今天和好奇心计划的发起人之一 —— Jeremy 开了一个会，会议的目的是讨论我们的方案是否值得投资，但是我认为更有价值的收获是他作为一个投资者对于“什么是好的产品”的分享。

什么是好奇心计划

由2016年在CUHKSZ读书的校友创立。他们许多在本科阶段就有自己的创业项目。
创立计划的目的是鼓励、找寻合适的投资的人，像YC (Y coordinate)一样进行产业孵化。

Jeremy的分享

他认为

  1. 目前市场上缺乏竞品的产品是垂类的AI agent，完成一个具体的任务，能执行长链路、多步骤操作，替代掉一些对于人来说是繁琐、重复性的工作。
  2. 技术人员应该勇于开启一个moonshot（大胆创新计划）,然后回推出一个解决方案

Jeremy分享了他个人作为Notion用户的经历，作为讨论 “什么是good taste of product”
他16年入学时恰逢Notion上线，他也作为第一批用户一直使用Notion到现在。他在和朋友创业的时候，在前期就试用了当时的的多数效率软件，最终选择了Notion，是因为Notion解决了某个痛点。
可能有点类似与obsidian之于我。

一个有good taste的产品，如何翻译这个taste？我的理解是解决市场痛点（有明确定位），解决方案优雅简介，交互设计注重细节。是三个从高层到底层的的层级。Jeremy在分享的时候提到了出处，但是我没记。

关于AI赋能的产品

Jeremy目前在做AI+区块链，之前做过科技+软背景镀金的一个产品，个人感觉他对product with AI feature是比较了解的（参见上文）
他分享了 —— 红杉 AI 峰会的链接
Software as a Tool
典型的Micosoft软件包，Vscode
Software as a Co-worker
cursor，Copilot等，协同完成任务
Software as an Outcome
帮你完成任务！

以及几个好产品的例子
granola是一个AI辅助的会议纪要软件，使用场合特定，目标人群画像精确。
星黎智能体，是一个陪伴型的AI，没找到过多的相关内容，可能受众小或者的已经跨了。

我的感触

做产品要靠独特的洞见，对于“优雅”的理解，做出最好的产品，在市场上获得一席之地，这很酷！
他提到要勇于提出问题：比如现在投简历很麻烦，你有不同的技能、项目经验，但是每一个岗位的侧重不一样，我应该如何让AI agent完成这一长条任务链。从信息整理，简历收集，到自动化简历投递。如果能让AI全线完成，将会是很好的市场。
我也在思考什么是好的产品：

  1. 让AI接入教学平台作为学习秘书是好的项目吗？我觉得是的，虽然道听途说的说谷歌也在做让AI接入bb，canvas之类的平台，但是也未尝不可自己做一份。
  2. 我之前思考过的多平台视频发布助手是一个好的项目，我很确定，如果可以很方便的接入每个平台的API进行内容分发管理，将会很抢手。
  3. Pixar台灯形状的互动机器人有搞头吗？我觉得一般，因为AI加硬件的项目很没有用武之地，交互的延时以及回答的质量还是太有问题，但是能不能让他做一些固定的任务，比如放在床头叫人起床、汇报天气之类

但是有些产品面对的对象没有明确的用户画像，像1&2我可以分别归到“使用这些平台的学生”和“经营多个平台的自媒体人”，但是3是没有明确的用户群体画像的，所以说不是一个能让我自信的产品。

这是我第一次感觉在龙大不后悔，不是因为住的舒服，而是因为周边的人，他们的眼界，他们的生活方式让我知道了什么是下一个社会阶层，我渴望实现这个跃迁。
